# Local Voice

> Community decision-making and voting.

---

| Property | Value |
|----------|-------|
| Category | Civic |

---

## Core Purpose

Give communities a way to make decisions together.

**Problems it solves:**
- Local decisions lack transparency
- Community members have little input
- Discussions rarely turn into action

---

## Key Features

### Proposals
- Anyone can propose
- Clear description
- Options to vote on
- Discussion period

### Discussion
- Threaded comments
- Pros and cons
- Questions and answers
- Expert input

### Voting
- Simple yes/no
- Multiple choice
- Ranked choice
- Budget allocation

### Transparency
- All votes visible
- Decision records
- Outcome tracking

### Delegation
- Delegate vote to trusted person
- Topic-specific delegates
- Revoke anytime

| Decision Type | Example |

| Elections | Choose garden coordinators |

---

## User Flow

```
1. Create proposal → Describe decision needed
2. Discussion period → Community discusses
3. Voting opens → Cast your vote
4. Voting closes → Results announced
5. Decision enacted → Record outcome
```

---

## Voting Methods

### Simple Yes/No
- Best for: Single decisions
- Example: "Should we plant tomatoes?"

### Multiple Choice
- Best for: Several options
- Example: "Which workshop should we host?"

### Ranked Choice
- Best for: Complex preferences
- Example: "Prioritize our projects"

### Budget Allocation
- Best for: Funding decisions
- Example: "How should we spend €500?"
