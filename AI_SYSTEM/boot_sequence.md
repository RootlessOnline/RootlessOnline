## Overview

Every AI (new or returning) must follow this boot sequence before doing any work. This ensures context consistency and prevents conflicts.

---

## Boot Sequence

### Step 1: Read Core Documents (Required)

Read in order:

| Order | File | Purpose | Time |
|-------|------|---------|------|
| 1 | START.md | Orientation | ~5 min |
| 2 | CONSTITUTION.md | Purpose + Principles | ~10 min |
| 3 | BYLAWS.md | Operational rules | ~10 min |
| 4 | STATE/current_state.md | Current status | ~2 min |
| 5 | STATE/decision_log.md | Previous decisions | ~5 min |

**Total time:** ~30 minutes

### Step 2: Confirm Understanding

After reading, AI should be able to answer:

1. What is The Collective building?
2. Who is Harley and how should I communicate with them?
3. What is my role and authority?
4. When should I escalate?
5. What is the current priority?

If any answer is unclear, ASK before proceeding.

### Step 3: Check Context Versions

Verify you have the correct versions:

| File | Current Version |
|------|-----------------|
| START.md | Check file header |
| CONSTITUTION.md | Check file header |
| BYLAWS.md | Check file header |

If versions don't match what you read before, **reload context**.

### Step 4: Check Current Work

Review STATE/current_state.md for:
- Active tasks
- Pending decisions
- Blockers
- Recent activity

### Step 5: Acknowledge Ready

Before starting work, confirm:
- "I have read START.md, CONSTITUTION.md, BYLAWS.md, current_state.md, and decision_log.md."
- "I understand my role and when to escalate."
- "I am ready to begin [assigned task]."

## For New AIs

Additional steps for first-time setup:

### Step N1: Register in AI_BOM.md

Provide the following information:

```markdown
### [Your Role] AI: [Your Name]

| Property | Value |
|----------|-------|
| **Name** | [Your identifier] |
| **Role** | [Your role] |
| **Provider** | [Your provider] |
| **Model** | [Your model] |
| **Access Method** | [How to reach you] |
| **Capabilities** | [What you can do] |
| **Limitations** | [What you cannot do] |
| **Contact** | [How to invoke you] |
| **Added** | [Today's date] |
| **Status** | Active |
```

### Step N2: Confirm Role in PERMISSIONS.yaml

Review your permissions in PERMISSIONS.yaml. Confirm:
- What decisions you can make
- What files you can read/write
- What git operations you can perform

### Step N3: Understand Coordination

If you are NOT Z:
- All your outputs go through Z
- You do not push to the repository directly
- You escalate conflicts to Z

If you ARE Z:
- You coordinate all other AIs
- You filter outputs before merge
- You escalate to Harley for principle conflicts

## For Returning AIs

If you've worked with The Collective before:

### Step R1: Quick Version Check

Have these files changed since your last session?
- START.md
- CONSTITUTION.md
- BYLAWS.md

If yes → Re-read changed files

### Step R2: Check Current State

What has happened since you were last active?
- Review recent entries in decision_log.md
- Review current_state.md
- Check for any flags or blockers

### Step R3: Resume or New Task

- Continue previous task if incomplete
- Start new task if assigned
- Check with Z if unsure

---

## Context Refresh Triggers

Reload context (re-read core documents) when:

1. **Explicitly asked** by Harley or Z
2. **Version mismatch** detected
3. **Conflict** with another AI occurred
4. **Extended absence** (>7 days since last work)
5. **Principle decision** was made (check decision_log.md)

## Output Standards

### Provenance Tag (Required)

Every AI output must include:

```markdown
---
Generated by: [AI Name]
Context versions: [file:version, file:version, ...]
Decisions referenced: [IDs or "none"]
Timestamp: [ISO 8601]
Reviewed by: [Z or "pending"]
---
```

```markdown
---
Generated by: Z
Context versions: START.md v1.0, CONSTITUTION.md v1.0, BYLAWS.md v1.0
Decisions referenced: D003
Timestamp: 2025-01-15T14:30:00Z
Reviewed by: pending
---
```
